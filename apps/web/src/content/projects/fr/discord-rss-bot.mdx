## À propos de Discord RSS Bot

Discord RSS Bot est un bot automatisé qui surveille les flux RSS et publie les mises à jour sur les canaux Discord via des webhooks. Entièrement configurable avec des flux et des catégories personnalisés, il dispose d'une gestion des délais d'attente pour éviter les blocages sur les flux RSS lents.

### Fonctionnalités clés

- **Surveillance automatique des flux RSS** avec des intervalles configurables
- **Mise en forme élégante pour Discord** avec des emojis de catégorie
- **Mise en cache intelligente** pour éviter les doublons
- **Gestion des délais d'attente** pour éviter les blocages sur les flux lents
- **Gestion robuste des erreurs** pour les problèmes réseau
- **Entièrement conteneurisé** avec support Docker
- **Images pré-construites** disponibles sur Docker Hub et GHCR
- **Multiplateforme** (AMD64, ARM64)
- **Configuration flexible** - flux universels ou basés sur des catégories
- **Plusieurs options de déploiement** (Docker, Service Système)
- **Journalisation complète** et vérifications de santé
- **Axé sur la sécurité** avec des conteneurs non-root

### Guide de démarrage rapide

#### Utilisation du conteneur pré-construit (recommandé)

```bash
# Create directories for persistent data/logs
mkdir -p ~/docker/docker-data/rss-discord-bot/data
mkdir -p ~/docker/docker-data/rss-discord-bot/logs

# Set permissions for container access (container runs as non-root)
chmod 777 ~/docker/docker-data/rss-discord-bot/logs
chmod 777 ~/docker/docker-data/rss-discord-bot/data

# Create .env file with your webhook URL (do not commit this file)
echo "DISCORD_WEBHOOK_URL=https://discord.com/api/webhooks/YOUR_ID/YOUR_TOKEN" > .env

# Run with Docker Compose (for testing with pre-built image)
docker compose -f docker-compose.test.yml up -d

# View logs
docker compose -f docker-compose.test.yml logs -f
```


#### Fedora/RHEL/CentOS


### Configuration

#### Variables requises

| Variable              | Description                       |
| --------------------- | --------------------------------- |
| `DISCORD_WEBHOOK_URL` | **Requis** : URL du webhook Discord |

#### Chemins (recommandés)

| Variable       | Défaut (image Docker)            | Description                                             |
| -------------- | -------------------------------- | ------------------------------------------------------- |
| `RSS_BOT_DATA` | `/app/data`                      | Répertoire où la base de données SQLite est stockée     |
| `RSS_BOT_LOGS` | `/app/logs`                      | Répertoire où les fichiers de log sont écrits          |
| `RSS_BOT_PID`  | `/tmp/discord-news-rss-bot.pid`  | Chemin du fichier PID (éviter `/var/run` dans les conteneurs non-root) |

#### Surcharges avancées (optionnelles)

| Variable   | Description                                             |
| ---------- | ------------------------------------------------------- |
| `DB_FILE`  | Chemin explicite du fichier de la base de données SQLite (remplace `RSS_BOT_DATA`) |
| `LOG_FILE` | Chemin explicite du fichier de log (remplace `RSS_BOT_LOGS`) |
| `DATA_DIR` | Ancien alias pour `RSS_BOT_DATA`                        |

#### Paramètres optionnels du botVoici la traduction en français :

---

| Variable             | Défaut | Description                        |
| -------------------- | ------ | ---------------------------------- |
| `CHECK_INTERVAL`     | 300    | Intervalle entre les vérifications (secondes) |
| `POST_DELAY`         | 3      | Délai entre les publications (secondes) |
| `COOLDOWN_DELAY`     | 60     | Délai après limitation de débit (secondes) |
| `MAX_POST_LENGTH`    | 1900   | Longueur maximale d'un message |
| `MAX_CONTENT_LENGTH` | 800    | Longueur maximale du contenu |
| `FEED_TIMEOUT`       | 30     | Délai d'expiration des requêtes RSS (secondes) |

### Configuration des flux RSS

Le bot prend en charge **deux méthodes de configuration** avec une gestion automatique des priorités :

#### Option 1 : Flux universels (simple)

Utilisez une seule variable pour tous vos flux RSS :

```bash
RSS_FEEDS=https://example.com/rss,https://another.com/feed,https://third.com/rss
```

#### Option 2 : Flux par catégorie (organisé)

Utilisez des variables distinctes pour différentes catégories :

```bash
RSS_FEEDS_NEWS=https://g1.globo.com/dynamo/rss2.xml,https://rss.uol.com.br/feed/noticias.xml
RSS_FEEDS_TECHNOLOGY=https://canaltech.com.br/rss/,https://tecnoblog.net/feed/
RSS_FEEDS_SPORTS=https://globoesporte.globo.com/rss/ultimas/
RSS_FEEDS_BUSINESS=https://www.infomoney.com.br/rss/
RSS_FEEDS_POLITICS=https://www.gazetadopovo.com.br/rss/brasil.xml
```

---

#### Système de priorité

1. **RSS_FEEDS** (universel) – prioritaire s'il est défini
2. **RSS_FEEDS_CATEGORY** (catégories) – utilisé s'il n'y a pas de flux universels
3. **Flux par défaut** – flux d'actualités brésiliennes en l'absence de configuration personnalisée

---

#### Personnalisation des flux

Vous pouvez personnaliser les flux de deux manières :

1. **Variables d'environnement (recommandé)**
   Utilisez le format `RSS_FEEDS_CATEGORY_NAME` comme décrit dans la section Configuration ci-dessus.

2. **Modification du code**
   Modifiez la fonction `parse_feeds_from_env()` dans `bot_service.py` pour changer les flux par défaut.
