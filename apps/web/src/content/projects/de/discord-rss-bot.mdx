## Über den Discord RSS Bot

Discord RSS Bot ist ein automatisierter Bot, der RSS-Feeds überwacht und Aktualisierungen über Webhooks an Discord-Kanäle sendet. Er ist vollständig konfigurierbar mit benutzerdefinierten Feeds und Kategorien und verfügt über eine Timeout-Behandlung, um Hängenbleiben bei langsamen RSS-Feeds zu verhindern.

### Hauptmerkmale

- **Automatische RSS-Überwachung** mit konfigurierbaren Intervallen
- **Elegantes Discord-Format** mit Kategorie-Emojis
- **Intelligentes Caching**, um doppelte Beiträge zu vermeiden
- **Timeout-Behandlung** verhindert Hängenbleiben bei langsamen Feeds
- **Robuste Fehlerbehandlung** für Netzwerkprobleme
- **Vollständig containerisiert** mit Docker-Unterstützung
- **Vorgefertigte Images** verfügbar auf Docker Hub & GHCR
- **Multi-Plattform** (AMD64, ARM64)
- **Flexible Konfiguration** – universelle oder kategorienbasierte Feeds
- **Mehrere Bereitstellungsoptionen** (Docker, Systemdienst)
- **Umfassende Protokollierung** und Gesundheitsprüfungen
- **Sicherheitsorientiert** mit Non-Root-Containern

### Schnellstart

#### Verwendung des vorgefertigten Containers (empfohlen)

```bash
# Create directories for persistent data/logs
mkdir -p ~/docker/docker-data/rss-discord-bot/data
mkdir -p ~/docker/docker-data/rss-discord-bot/logs

# Set permissions for container access (container runs as non-root)
chmod 777 ~/docker/docker-data/rss-discord-bot/logs
chmod 777 ~/docker/docker-data/rss-discord-bot/data

# Create .env file with your webhook URL (do not commit this file)
echo "DISCORD_WEBHOOK_URL=https://discord.com/api/webhooks/YOUR_ID/YOUR_TOKEN" > .env

# Run with Docker Compose (for testing with pre-built image)
docker compose -f docker-compose.test.yml up -d

# View logs
docker compose -f docker-compose.test.yml logs -f
```

### RSS-Feeds-Konfiguration

Der Bot unterstützt **zwei Konfigurationsmethoden** mit automatischer Prioritätsverarbeitung:

#### Option 1: Universelle Feeds (Einfach)

Verwenden Sie eine einzelne Variable für alle Ihre RSS-Feeds:

```bash
RSS_FEEDS=https://example.com/rss,https://another.com/feed,https://third.com/rss
```

#### Option 2: Kategorienbasierte Feeds (Organisiert)

Verwenden Sie separate Variablen für verschiedene Kategorien:

```bash
RSS_FEEDS_NEWS=https://g1.globo.com/dynamo/rss2.xml,https://rss.uol.com.br/feed/noticias.xml
RSS_FEEDS_TECHNOLOGY=https://canaltech.com.br/rss/,https://tecnoblog.net/feed/
RSS_FEEDS_SPORTS=https://globoesporte.globo.com/rss/ultimas/
RSS_FEEDS_BUSINESS=https://www.infomoney.com.br/rss/
RSS_FEEDS_POLITICS=https://www.gazetadopovo.com.br/rss/brasil.xml
```

---

#### Prioritätssystem

1. **RSS_FEEDS** (universell) – hat Priorität, falls gesetzt
2. **RSS_FEEDS_CATEGORY** (Kategorien) – wird verwendet, falls keine universellen Feeds vorhanden sind
3. **Standard-Feeds** – brasilianische Nachrichtenfeeds, falls keine benutzerdefinierte Konfiguration vorhanden ist

### Entwicklung

#### Lokal ausführen (ohne Docker)

```bash
# Install dependencies
pip install -r requirements.txt

# Configure environment variables
export DISCORD_WEBHOOK_URL="your_webhook_url_here"

# Run the bot
python bot_service.py
```

---

#### Feeds anpassen

Sie können die Feeds auf zwei Arten anpassen:

1. **Umgebungsvariablen (Empfohlen)**
   Verwenden Sie das Format `RSS_FEEDS_CATEGORY_NAME`, wie im Abschnitt Konfiguration oben beschrieben.

2. **Code-Änderung**
   Bearbeiten Sie die Funktion `parse_feeds_from_env()` in `bot_service.py`, um die Standard-Feeds zu ändern.
