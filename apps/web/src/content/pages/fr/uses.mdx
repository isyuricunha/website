Voici un inventaire complet du matériel qui alimente mon homelab, mon travail de développement et mon informatique quotidienne.

Pas de marketing, pas de liens d'affiliation — juste les machines que j'utilise réellement et pourquoi elles font partie de ma configuration.

Le matériel est organisé en :

- **ordinateur portable personnel** — mon outil de travail quotidien
- **serveur principal du homelab** — la machine puissante pour les bases de données et les services
- **nœud ARM64 du homelab** — Raspberry Pi pour les tests ARM
- **machines x86-64 dans le cloud** — dev et prod dans le cloud
- **machines ARM64 dans le cloud** — charges de travail ARM dans le cloud

---

## Matériel personnel

### Ordinateur portable principal

C'est mon outil de travail quotidien. Je l'utilise pour tout ce qui concerne l'étude, le code, SSH, les documents et le travail général.

**Caractéristiques**

- CPU
  - **AMD Ryzen 7 7445HS**
- RAM
  - **64 Go DDR5**
- Stockage
  - **256 Go** SSD
  - **360 Go** SSD
  - utilisé pour le système, les outils de développement, les bases de données locales, les conteneurs et les fichiers
- GPU
  - **Nvidia RTX 4070** (portable)

Principaux cas d'utilisation :

- terminaux et IDE pour travailler sur le code et la documentation
- connexion SSH à toutes les machines du labo et du cloud via Tailscale
- clients de bases de données, outils d'administration et tableaux de bord
- navigateurs et applications qui communiquent avec les services exécutés dans le labo
- quelques conteneurs locaux et VM légères lorsque nécessaire

Cet ordinateur portable est l'endroit d'où je contrôle tout le reste.

---

## Matériel du homelab

### Serveur principal à la maison

C'est la grosse machine du homelab, axée sur les bases de données, les services et les expériences.

**CPU**

- **Intel Xeon W9 3475X**

**Mémoire**

- **256 Go ECC R**
- **256 Go ECC R** en configuration miroir
- un total de **512 Go** de mémoire ECC, configurée pour la redondance afin d'éviter les pannes

**Stockage**

Stockage rotatif :

- **14 To HDD** pour les données principales
- **14 To HDD** pour la redondance
- **14 To HDD** pour les sauvegardes

Stockage NVMe :

- **256 Go NVMe** pour le système
- **500 Go NVMe** utilisé comme cache de fichiers
- **4 To NVMe** pour les fichiers froids et le stockage à long terme

**GPU**

- **Nvidia GT210** pour la sortie vidéo
- **Nvidia GT 1050** pour les petites charges de travail et les codecs vidéo
- **AMD Radeon RX 7900 XTX** pour les charges de travail IA

> Remarque : je prévois de remplacer tout le matériel Nvidia et Intel par des composants AMD à l'avenir

**Utilisation typique :**

- PostgreSQL et autres bases de données dans des configurations de labo réalistes
- Piles Docker/Conteneurs pour les services auto-hébergés
- Surveillance, journalisation et tâches de sauvegarde
- Expériences d'IA utilisant la RX 7900 XTX
- Stockage pour les médias, les sauvegardes et les données du labo avec redondance

C'est le cœur du homelab et l'endroit où se déroule la plupart des travaux intensifs.

---

### Nœud ARM64 dans le homelab

Je possède également un environnement ARM64 à la maison pour les tests et la diversité.

**Carte**

- **Raspberry Pi 5**, version 16 Go

**SoC**

- **Broadcom BCM2712**
- quad core **ARM Cortex A76**

**Mémoire**

- **16 Go LPDDR4X 4267**

**Stockage**

- Module **64 Go eMMC** pour le système
- Disque **512 Go** pour les conteneurs et les fichiers

Cas d'utilisation :

- Exécution de conteneurs ARM
- Test des services sur ARM64 avant de les déployer sur le cloud ARM
- Petits services toujours actifs avec une faible consommation d'énergie
- Expériences de stockage sur de petites cartes

Ce nœud m'aide à m'assurer que mes configurations ne sont pas uniquement x86.

## Matériel cloud

J'exécute également des charges de travail de développement et de production dans le cloud. Actuellement, il y a deux principaux groupes de VM.

### Machines x86 64 dev et prod dans le cloud

Ce sont deux machines virtuelles avec le même matériel, une pour le développement et une pour la production.

**Caractéristiques par VM**

- CPU
  - **AMD EPYC 7551**, **8 vCPU**
- RAM
  - **16 Go**
- Stockage
  - **250 Go** pour le système, les données et les logs

Il y a **deux** de ces machines avec la même configuration.

Utilisation principale :

- Exécution de bases de données et de services dans un environnement plus réaliste de type production
- Test des déploiements qui communiquent avec le homelab via Tailscale
- Simulation de scénarios où une partie de la pile est sur site et une autre partie dans le cloud

---

### Machines ARM64 dev et prod dans le cloud

Je possède également des machines ARM64 dans le cloud pour les tests et les charges de travail qui ont du sens sur cette architecture.

**Caractéristiques par VM**

- CPU
  - **8 vCPU**
- RAM
  - **24 Go**
- Stockage
  - **150 Go** pour le système et les données> **Remarque** : le modèle exact du CPU n'est pas spécifié par le fournisseur cloud - ce qui compte, ce sont les performances et l'architecture ARM64.

**Utilisé pour :**

- exécuter des conteneurs et services ARM64
- vérifier les différences de performance entre les configurations ARM et x86
- expérimenter des charges de travail écoénergétiques

Encore une fois, l'idée est d'avoir au moins les environnements de développement et de production séparés dans ce groupe.

---

## Pourquoi ce mélange de matériel

Le matériel n'est pas choisi au hasard. Il suit une idée simple.

- le notebook est le centre de contrôle
- le serveur principal du labo maison est l'endroit où se trouvent les travaux lourds et le stockage
- le nœud ARM64 maison me permet de varier les expériences et de travailler avec une faible consommation d'énergie
- les machines cloud me permettent de simuler des configurations de production plus réalistes

Avec cette combinaison, je peux :

- tester des bases de données et des services avec une mémoire et un stockage sérieux à la maison
- exécuter des charges de travail d'IA localement en utilisant la RX 7900 XTX
- maintenir la redondance et les sauvegardes avec une RAM en miroir et plusieurs disques durs
- comparer le comportement des architectures x86 et ARM64
- pratiquer la séparation des environnements de développement, de pré-production et de production
