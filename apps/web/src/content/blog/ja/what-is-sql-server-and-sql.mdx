# SQL Serverデータベースとは何ですか？

SQL Serverデータベースは、Microsoftが開発したリレーショナルデータベース管理システムです。構造化データと非構造化データを含む大量のデータを格納・管理するために使用されます。SQL Serverは、格納されたデータのセキュリティ、可用性、整合性を確保し、システムのパフォーマンスとスケーラビリティを保証するための幅広い機能を提供しています。

SQL Server Express、SQL Server Standard、SQL Server Enterpriseなど、いくつかのバージョンのSQL Serverが利用可能です。これらの各バージョンは異なる機能セットを提供し、異なるタイプの用途に適しています。

SQL Serverを使用するには、構造化クエリ言語（SQL）を知る必要があります。SQLはリレーショナルデータベースを操作するための標準言語であり、データベースに格納されたデータの作成、クエリ、変更に使用されます。

<br />

# 構造化クエリ言語（有名なSQL）とは何ですか？

構造化クエリ言語（SQL）は、リレーショナルデータベースに格納されたデータを管理・操作するために使用されるプログラミング言語です。これはリレーショナルデータベースを操作するための標準言語であり、データベースに格納されたデータの作成、クエリ、変更に使用されます。

SQLは宣言型言語で、これは何をする必要があるかを記述するために使用され、どのようにすべきかを指定するのではないことを意味します。これにより、データベース管理システム（DBMS）がSQLクエリの実行を最適化し、システムリソースの効率的な使用を確保できます。

SQLは、テーブルとインデックスの作成・変更、データの挿入・更新、データベースからのデータの取得など、幅広いタスクを実行するために使用されます。また、データベース内のデータの構造と関係を定義し、ユーザーの権限と制御を設定するためにも使用されます。

SQLは、ビジネス、金融、ヘルスケア、政府など、さまざまな業界で広く使用されている強力で汎用性の高い言語です。データベース管理者と開発者にとって不可欠なツールであり、データアナリストやデータサイエンティストが分析とレポート目的でデータを抽出、変換、ロードするためにも使用されます。

MySQL、Oracle、Microsoft SQL Serverなど、SQLにはいくつかの方言があり、それぞれ独自の機能と構文を持っています。しかし、SQLの核となる概念と原則はすべての方言で同じであり、学習しやすく広く適用可能な言語となっています。

# SQLクエリの例

SQL Serverデータベースで実行できるSQLクエリの例をいくつか紹介します：

## _Select クエリ_

- 以下のクエリは、「City」フィールドが「São Paulo」と等しい「Customers」テーブルからすべてのフィールドを選択します：

```sql
SELECT * FROM Customers WHERE City = 'São Paulo';
```

<br />

## _データの挿入_

- 以下のクエリは、「Customers」テーブルに新しいレコードを挿入します：

```sql
INSERT INTO Customers (Name, Address, City) VALUES ('João da Silva', 'Rua das Flores, 123', 'São Paulo');
```

<br />

## _データの更新_

- 以下のクエリは、「Customers」テーブルのID 1のレコードの「Address」フィールドを更新します：

```sql
UPDATE Customers SET Address = 'Rua das Oliveiras, 456' WHERE ID = 1;
```

<br />

## _データの削除_

- 以下のクエリは、「Customers」テーブルからID 1のレコードを削除します：

```sql
DELETE FROM Customers WHERE ID = 1;
```

# 2つのシンプルな例

<br />

## 最初の例

<br />

> あなたは小さな顧客データベースのデータベース管理者です。以下の情報で新しい顧客レコードを追加する必要があります：

<br />

- 名前：「John Smith」
- 住所：「123 Main Street」
- 市：「New York」

<br />
<br />

## _解決策_

<br />

- 新しい顧客レコードを追加するには、以下のSQLクエリを使用できます：

```sql
INSERT INTO Customers (Name, Address, City) VALUES ('John Smith', '123 Main Street', 'New York');
```

<br />

_このクエリは、指定された名前、住所、市で「Customers」テーブルに新しいレコードを挿入します。_

<br />
<br />

## 2番目の例

<br />

> あなたは製品在庫データベースのデータベース管理者です。ID「12345」の製品の数量を10に更新する必要があります。

<br />
<br />

## _解決策_

<br />

- 製品在庫データベースのデータベース管理者として、ID「12345」の製品の数量を10に更新する必要があります。

```sql
UPDATE Products SET Quantity = 10 WHERE ID = 12345;
```

<br />

_このクエリは、ID「12345」の製品の「Quantity」フィールドを値10に更新します。_

# 複雑な例

SQL Serverデータベースを含む複雑な例と解決策をご紹介します：

> あなたは大手オンライン小売業者のデータベース管理者です。会社には、顧客、注文、製品に関する情報を格納するいくつかのテーブルを持つデータベースがあります。このデータベースは、顧客情報の追跡、注文の処理、レポートの生成に使用されます。

> 会社の営業マネージャーの一人が、各顧客の総売上と注文数を製品カテゴリ別に分けて表示するレポートを要求しました。レポートは過去1年間のデータを表示し、総売上の降順でソートする必要があります。

<br />
<br />

## _解決策_

<br />

- 要求されたレポートを生成するには、以下のSQLクエリを使用できます：

```sql
SELECT c.Name AS 'Customer',
       SUM(o.Total) AS 'Total Sales',
       COUNT(o.ID) AS 'Number of Orders',
       p.Category AS 'Product Category'
FROM Customers c
INNER JOIN Orders o ON c.ID = o.CustomerID
INNER JOIN Products p ON o.ProductID = p.ID
WHERE o.Date BETWEEN '2022-01-01' AND '2022-12-31'
GROUP BY c.Name, p.Category
ORDER BY 'Total Sales' DESC;
```

<br />

_このクエリは、指定された日付範囲内に配置されたすべての注文について、顧客名、総売上、注文数、製品カテゴリを選択します。その後、結果を顧客名と製品カテゴリでグループ化し、総売上の降順で結果をソートします。結果のレポートは、各顧客の総売上と注文数を製品カテゴリ別に分けて表示します。_
