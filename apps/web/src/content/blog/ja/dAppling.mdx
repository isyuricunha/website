# はじめに

分散型ウェブは急速に成長しているものの、フロントエンドを検閲耐性かつ安全にホストする方法を見つけることは、開発者にとって大きな課題のひとつです。dAppling Network は、分散型フロントエンドをシンプルかつ使いやすくホスティングするための新しいプラットフォームとして、この課題の解決を目指します。

# dAppling Network とは？

dAppling Network は Web3 プロトコル向けに特化したホスティングサービスです。フロントエンドのコードは InterPlanetary File System (IPFS) 上に保存・配信されるため、検閲耐性と分散性が得られます。さらに dAppling は次のような、Web3 開発者にとって価値ある機能を提供します。

- **検証可能なデプロイ:** フロントエンドが正しくデプロイされ、改ざんされていないことを検証可能なプロセスで保証します。
- **スマートコントラクト制御のフロントエンド:** スマートコントラクトでフロントエンドを制御でき、アプリケーションのコントロール性が高まります。
- **パフォーマンス:** IPFS によりフロントエンドを配信し、高いパフォーマンスを実現します。

# どのように動くのか

dAppling を使うには、アカウントを作成してフロントエンドのコードをアップロードするだけです。dAppling がコードを IPFS にデプロイし、アクセス用の URL を生成します。

# なぜ dAppling Network を使うのか

フロントエンドのホスティングに dAppling Network を選ぶ理由はいくつもあります。

- **検閲耐性:** IPFS に保存・配信するため、政府や組織による検閲から保護されます。
- **セキュリティ:** 検証可能なデプロイやスマートコントラクト制御といった機能で前面を保護します。
- **パフォーマンス:** IPFS による配信で高速です。
- **使いやすさ:** 数分でアカウント作成とフロントエンドのデプロイが完了します。

# dAppling Network は継続的に進化している

dAppling Network は急速に発展しており、チームは常に改善に取り組んでいます。主なフォーカス領域は次のとおりです。

- **セキュリティ:** 最高水準のセキュリティを目指し、新機能の導入や既存対策の強化を継続しています。
- **パフォーマンス:** キャッシュ機構の改善やコード最適化など、継続的に最適化を実施しています。
- **機能:** ユーザーのフィードバックを取り入れ、コミュニティが求める新機能を追加しています。

# よくある質問 (FAQ)

**Q: dAppling Network を他のホスティングと比べた利点は？**

A: 次の利点があります。

- **検閲耐性:** IPFS を用いるため、フロントエンドは検閲されにくい構造です。
- **セキュリティ:** 検証可能なデプロイ、スマートコントラクト制御などで防御。
- **パフォーマンス:** IPFS 配信により高速。
- **使いやすさ:** 数分でデプロイ可能。

**Q: 制約はありますか？**

A: なお開発中のプラットフォームであり、従来型ホスティングにあるすべての機能を現時点で網羅しているわけではありません。とはいえ、継続的な改善により、この種の制約は順次解消される見込みです。

**Q: 料金は？**

A: 基本アカウントは無料です。プレミアム機能は有料で、料金はホストするフロントエンド数などに依存します。

**Q: 使い始めるには？**

A: アカウントを作成し、フロントエンドのコードをアップロードするだけで、IPFS にデプロイされ、アクセス用 URL が付与されます。

**Q: 技術要件は？**

A: ウェブブラウザのみで利用できます。あらゆる Web3 プロトコルのフロントエンドをホスティング可能です。
